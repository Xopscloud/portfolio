## Project Overview

Developed a comprehensive construction management platform using React frontend and Java Spring Boot backend, fully containerized with Docker and deployed using GitHub Actions with Kubernetes orchestration for scalable project management.

## Application Architecture

### Frontend (React)
- **Framework**: React 18 with TypeScript
- **State Management**: Redux Toolkit
- **UI Components**: Material-UI with custom themes
- **Features**: Project tracking, resource management, real-time updates

### Backend (Java Spring Boot)
- **Framework**: Spring Boot 3.0 with Java 17
- **Architecture**: Microservices with Spring Cloud
- **Database**: PostgreSQL with JPA/Hibernate
- **Security**: Spring Security with JWT authentication

### Microservices Design
- **Project Service**: Project lifecycle management
- **Resource Service**: Equipment and material tracking
- **User Service**: Authentication and authorization
- **Notification Service**: Real-time alerts and updates

## Containerization Strategy

### Frontend Dockerfile
```dockerfile
# Multi-stage build for React app
FROM node:18-alpine as build
WORKDIR /app
COPY package*.json ./
RUN npm ci --only=production
COPY . .
RUN npm run build

FROM nginx:alpine
COPY --from=build /app/build /usr/share/nginx/html
COPY nginx.conf /etc/nginx/nginx.conf
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

### Backend Dockerfile
```dockerfile
FROM openjdk:17-jdk-slim
VOLUME /tmp
ARG JAR_FILE=target/*.jar
COPY ${JAR_FILE} app.jar
ENTRYPOINT ["java","-jar","/app.jar"]
```

## CI/CD with GitHub Actions

### Workflow Configuration
```yaml
name: CI/CD Pipeline

on:
  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main ]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v3
    
    - name: Set up JDK 17
      uses: actions/setup-java@v3
      with:
        java-version: '17'
        distribution: 'temurin'
    
    - name: Run tests
      run: ./mvnw test
    
    - name: Build with Maven
      run: ./mvnw clean package
    
    - name: Build Docker image
      run: docker build -t construction-app:${{ github.sha }} .
```

### Deployment Pipeline
```yaml
  deploy:
    needs: test
    runs-on: ubuntu-latest
    if: github.ref == 'refs/heads/main'
    
    steps:
    - name: Deploy to Kubernetes
      uses: azure/k8s-deploy@v1
      with:
        manifests: |
          k8s/deployment.yaml
          k8s/service.yaml
        images: |
          construction-app:${{ github.sha }}
```

## Kubernetes Orchestration

### Deployment Configuration
```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: construction-app
spec:
  replicas: 3
  selector:
    matchLabels:
      app: construction-app
  template:
    metadata:
      labels:
        app: construction-app
    spec:
      containers:
      - name: backend
        image: construction-backend:latest
        ports:
        - containerPort: 8080
        env:
        - name: SPRING_PROFILES_ACTIVE
          value: "production"
      - name: frontend
        image: construction-frontend:latest
        ports:
        - containerPort: 80
```

### Helm Chart Structure
```yaml
# values.yaml
replicaCount: 3

image:
  repository: construction-app
  tag: latest
  pullPolicy: IfNotPresent

service:
  type: LoadBalancer
  port: 80
  targetPort: 8080

ingress:
  enabled: true
  annotations:
    kubernetes.io/ingress.class: nginx
    cert-manager.io/cluster-issuer: letsencrypt-prod
  hosts:
    - host: construction.example.com
      paths:
        - path: /
          pathType: Prefix
```
